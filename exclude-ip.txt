# 以下のIPは「スキャン」と判定したアクセス元で、集計・ログから除外
# 判定は次の5種(A/B/C/S/L)のいずれかに該当（しきい値は env.ps1 で調整可）
# 共通ゲート（A/B/C にのみ適用。S/L は対象外）: 総リクエスト数が一定数（既定30）未満、または
#     最も混雑した窓（既定1分）の1分あたり要求数（ピークレート）が一定（既定10件/分）未満のIPは A/B/C の対象外
# (A) 列挙スキャン: 異なるURLパスが一定数（既定20）以上、かつ
#     404が一定数（既定20）以上、または200応答が全体の一定割合（既定1割）未満
# (B) 高エラー型: 401/403/404/429（4xx）に5xxを重み（既定0.5）で加味したエラー量が一定数（既定30）以上、かつ
#     その値が総応答の一定割合（既定5割）以上
# (C) クレデンシャル総当たり: 経路1=単一ログインフォームへのPOSTが一定数（既定20）以上、または
#     経路2=単一パスへのアクセスが一定数（既定50）以上、かつ そのパスがログインフォーム一致 または 同一パス上の401+403+429が一定数（既定20）以上
#     ※集中パスの判定はログインフォーム（既定で /login・/signin・/user/login・/admin/login・wp-login 等。/api/* や /graphql は含めない）
# (S) 内容指標: リクエスト内容が攻撃のもの（共通ゲート免除・1回でも該当で検知）
#     攻撃ファイルパス（.env・.git・wp-login.php・/cgi-bin/..traversal・/actuator 等）／攻撃トークン（../../・union select・${jndi:・<script 等。クエリ含む）／
#     絶対URI（プロキシ探索）／異常メソッド（CONNECT 等）／確定プロトコルシグネチャ（SMB・Gh0st・mining 等）または整形不正バイナリが支配的
# (L) ログイン探索: 異なるログインページの種類数が一定数（既定3）以上、かつ
#     認証パス上の200成功率が一定割合（既定5割）未満（共通ゲート免除）
